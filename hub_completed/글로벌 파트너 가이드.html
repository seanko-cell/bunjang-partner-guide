주문 상태 흐름Order Status Flow

번개장터의 전체 주문 상태 도식입니다. 파트너사 운영 시 참고하세요.

Full order status flow diagram for Bunjang. Please refer to this when managing operations.

번개장터 주문상태 도식 (ver. 2025-05)Bunjang Order Status Diagram (ver. 2025-05)

1. 한국 배송지 필수Korean Shipping Address Required

주문 API 호출 시 반드시 한국 내 배송지 주소를 포함해야 합니다.

When calling the Order API, a Korean domestic shipping address must be included in the request.

  • 파트너사의 한국 내 창고 또는 검수 거점 주소를 배송지로 등록Register the partner's Korean warehouse or inspection facility as the shipping address
  • 해외 주소 입력 시 주문 생성 불가Orders cannot be created with overseas addresses

2. 구매확정 정책Order Confirmation Policy

자동 구매확정: 배송 완료 후 5일 (자연일 기준)

Auto-confirmation: 5 calendar days after delivery

  • 기본 정책(3일)에서 파트너사 검수 시간을 고려하여 5일로 연장 적용Extended from the standard 3-day policy to 5 days to accommodate partner inspection time
  • 구매확정 후 판매자에게 대금 정산이 이루어짐Payment is settled to the seller upon order confirmation
✅ 권장: 상품 도착 후 2일 이내 구매확정 API 실행Recommended: Call Confirm Order API within 2 days of delivery
구매확정이 지연될 경우 판매자가 다음 거래를 거절하는 사례가 발생할 수 있습니다. 빠른 확정을 권장합니다. Delayed confirmation may cause sellers to decline future transactions. Prompt confirmation is strongly recommended.
⚠️ 배송 수단 안내Delivery Method Note
API를 통해 주문 시 일반 택배만 선택 가능합니다. 알뜰택배(편의점 간 배송)가 필요한 경우, 판매자와 직접 소통하여 진행해야 하며 API에서는 선택 불가입니다. Only standard courier delivery is available via API. If economy delivery (convenience store-to-store) is needed, it must be arranged directly with the seller — it cannot be selected through the API.

3. 멀티 상품 리스팅 주의Multi-Product Listing Caution

번개장터는 상품 옵션 선택 기능이 없습니다. 일부 판매자는 하나의 상품 리스팅에 여러 상품을 묶어서 판매합니다.

Bunjang does not support product option selection. Some sellers list multiple items under a single product listing.

  • 이런 경우 구매 전 판매자와 CS를 통해 원하는 상품을 특정해야 합니다In such cases, the specific desired item must be confirmed with the seller via CS before purchasing
  • CS 없이 주문 시 원하지 않는 상품이 발송될 수 있습니다Ordering without CS may result in receiving an unintended item

예시: 하나의 리스팅에 여러 상품이 포함된 경우Example: Single listing with multiple items

예시: 판매자와 CS를 통해 상품 특정 후 구매 진행Example: Confirming item with seller via CS

5. 포인트 충전 및 구매 방식Point Charging & Purchase Flow

번개장터에서의 모든 구매는 포인트를 통해 이루어집니다.

All purchases on Bunjang are made using points.

1
포인트 충전 요청 API 호출Call Point Charge API POST /api/v1/points/issuance
응답값으로 가상계좌 번호를 수신 A virtual account number is returned in the response
2
가상계좌 입금Deposit to Virtual Account 발급된 계좌에 입금하면 입금액 그대로 파트너 계정에 포인트로 자동 충전됩니다. Upon deposit to the issued account, the exact amount is automatically credited as points to the partner account.
3
포인트로 상품 구매Purchase Products with Points 충전된 포인트를 사용하여 주문 생성 API를 통해 구매 진행 Use the charged points to create orders via the Order API
⚠️ 유저당 READY 상태 충전 요청은 1건만 유지 가능합니다. FAIL 건은 재사용 불가합니다. Only 1 READY charge request is allowed per user at a time. Failed requests cannot be reused.
